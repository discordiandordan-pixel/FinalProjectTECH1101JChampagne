1. Variables — var, let, const

Three ways to declare variables. Prefer const by default, use let when the value needs to change, and avoid var (function-scoped and hoisted in confusing ways).

If you've used C++, const in JavaScript works similarly to const std::string — it locks the binding so the value can't be reassigned. This keeps your code more modular and reduces clutter in the main logic, the same way declaring constants outside of main() in C++ helps keep things clean and readable.

const name = "Jordan";   // cannot be reassigned
let score = 0;           // can be reassigned
var old = true;          // avoid — function-scoped

2. Data Types

JavaScript has 7 primitive types plus objects. Unlike C++, you don't declare the type when creating a variable — JavaScript figures it out at runtime (it's dynamically typed). In C++ you'd write int x = 42; or std::string name = "Jordan";, but in JavaScript it's just let x = 42; regardless of type. Use typeof to check what type a value is at any point.

typeof "hello"      // "string"
typeof 42           // "number"
typeof true         // "boolean"
typeof undefined    // "undefined"
typeof null         // "object" (quirk!)
typeof Symbol()     // "symbol"
typeof 9n           // "bigint"
typeof {}           // "object"
typeof []           // "object"

3. Functions

Functions in JavaScript work similarly to C++ functions — they take parameters, do work, and return a value. The main differences: no return type is declared (JavaScript is dynamically typed), and you have a shorthand called an arrow function (=>) which is similar in spirit to a C++ lambda expression ([](){}). Arrow functions are great for short, one-liner operations.

// Declaration
function greet(name) 
{
    return "Hello, " + name;
}

// Arrow function (concise)
const greet = (name) => "Hello, " + name;

// Call it
console.log(greet("Jordan")); // Hello, Jordan

4. if / else / else if

Conditional logic works almost identically to C++. The syntax — if (condition) { } — is the same. The main difference is that JavaScript is more relaxed about what counts as "true" or "false" (called truthy/falsy). For example, an empty string "" or 0 is falsy in JavaScript, similar to how 0 evaluates to false in a C++ boolean context.

const temp = 15;

if (temp > 25) {
    console.log("Hot day");
} else if (temp >= 10) {
    console.log("Nice day");
} else {
    console.log("Cold day");
}

5. for Loop

The for loop syntax is identical to C++ — for (init; condition; increment). If you're comfortable with C++ loops, this is a direct copy. One difference: use let instead of int for the loop counter since JavaScript doesn't have typed declarations.

for (let i = 0; i < 5; i++) {
    console.log("Count: " + i);
}
// Prints Count: 0 through Count: 4

6. while Loop

The while loop is also identical in structure to C++. Same rule applies: make sure the condition eventually becomes false or you'll get an infinite loop — just like in C++. A common use case is waiting for user input or polling a condition, similar to a control loop in a PLC or automation context.

let count = 0;
while (count < 3) {
    console.log(count);
    count++;
}
// Prints 0, 1, 2

7. Arrays

JavaScript arrays are similar to C++ std::vector — they're dynamic (no fixed size), zero-indexed, and can grow or shrink. Unlike C++ arrays where you'd declare a type like int arr[5], JavaScript arrays can hold mixed types and resize automatically. push() is like push_back() in a vector, and pop() removes the last element just like pop_back().

const cats = ["Luna", "Mochi", "Pixel"];
console.log(cats[0]);       // "Luna"
cats.push("Noodle");        // add to end
cats.pop();                 // remove last
console.log(cats.length);   // 3

8. Objects

JavaScript objects are the closest thing to a C++ struct — they group related data under one name using key-value pairs. Where in C++ you'd define a struct with typed members (struct Cat { string name; int age; };), in JavaScript you just write it inline with no type declarations. Objects can also hold functions as values (called methods), making them similar to a simple C++ class as well.

const cat = {
    name: "Luna",
    age: 4,
    isSpayed: true
};

console.log(cat.name);       // "Luna"
console.log(cat["age"]);     // 4

9. DOM Selection

The DOM (Document Object Model) is how JavaScript interacts with the HTML on the page — there's no direct equivalent in C++ since C++ doesn't run in a browser. Think of it like getting a pointer or reference to an HTML element so you can read or modify it. getElementById is the most direct — like looking up a variable by its exact name. querySelector is more flexible, accepting any CSS selector.

// By ID
const btn = document.getElementById("myBtn");

// By CSS selector (first match)
const title = document.querySelector("h1");

// All matches
const items = document.querySelectorAll(".nav-link");

10. Event Listeners

Event listeners are JavaScript's way of responding to user actions — there's no direct C++ equivalent unless you've worked with GUI frameworks or interrupt-driven systems (which is actually similar in concept to hardware interrupts in PLC/automation programming). Instead of polling in a loop to check if something happened, you register a function that fires automatically when the event occurs — just like an interrupt service routine (ISR).

const btn = document.getElementById("myBtn");

btn.addEventListener("click", function() {
    alert("Button clicked!");
});

// Arrow function version
btn.addEventListener("click", () => alert("Clicked!"));

11. Changing DOM Content & Style

Once you have a reference to an HTML element (see Entry 9), you can read or change almost anything about it. This is unique to JavaScript in the browser — C++ has no equivalent since it doesn't render HTML. Think of it like directly writing to a register or output in a PLC: you grab the reference, then set the value. textContent is safe for plain text; innerHTML lets you inject raw HTML but use it carefully to avoid security issues.

const heading = document.querySelector("h1");

heading.textContent = "New Title";       // change text
heading.innerHTML = "<em>Italic Title</em>"; // change HTML

heading.style.color = "red";             // inline style
heading.classList.add("highlight");      // add CSS class
heading.classList.remove("highlight");   // remove CSS class
heading.classList.toggle("active");      // toggle CSS class

12. Template Literals

Template literals replace messy string concatenation with a cleaner embedded syntax. In C++ you'd use std::cout << "Hi " << name << ", score: " << score; which gets verbose fast. JavaScript's template literals use backticks and ${} to embed variables directly in the string — much more readable, especially for building dynamic HTML or messages.

const name = "Jordan";
const score = 95;

// Old way
console.log("Hi " + name + ", your score is " + score);

// Template literal
console.log(`Hi ${name}, your score is ${score}`);

13. Array Methods — forEach, map, filter

These are higher-order functions built into JavaScript arrays — there's no direct equivalent in basic C++, though the C++ Standard Library has similar algorithms (std::for_each, std::transform, std::copy_if). They let you process arrays without writing manual for loops, which keeps code shorter and easier to read. forEach just iterates, map transforms and returns a new array, and filter returns only items that pass a test.

const nums = [1, 2, 3, 4, 5];

// forEach — run code for each item
nums.forEach(n => console.log(n));

// map — transform each item, return new array
const doubled = nums.map(n => n * 2); // [2,4,6,8,10]

// filter — keep items that pass the test
const evens = nums.filter(n => n % 2 === 0); // [2,4]

14. console.log & Debugging

In C++ you'd use std::cout << value; to print output for debugging. In JavaScript, console.log() is the equivalent — it prints to the browser's DevTools console (press F12 to open it). console.warn and console.error add colour-coded severity levels. The debugger statement is like setting a breakpoint in Visual Studio — execution pauses there so you can inspect variables step by step.

console.log("Hello");           // print a value
console.warn("Watch out!");     // yellow warning
console.error("Something broke!"); // red error
console.table([{a:1},{a:2}]);   // table view

// Pause execution in DevTools
debugger;

15. Ternary Operator

The ternary operator exists in C++ too — the syntax is identical: condition ? valueIfTrue : valueIfFalse. It's a compact one-liner that replaces a simple if/else. Both languages use it the same way, so if you've written it in C++ it works exactly the same in JavaScript. Great for assigning a value based on a condition without the extra lines of a full if/else block.

const age = 20;
const status = age >= 18 ? "adult" : "minor";
console.log(status); // "adult"

// Equivalent if/else:
// if (age >= 18) { status = "adult"; } else { status = "minor"; }

16. Null & Undefined

C++ has nullptr for pointers that point to nothing. JavaScript has two similar concepts: null (intentionally set to "no value") and undefined (declared but never assigned — JavaScript's version of an uninitialised variable). In C++ an uninitialised variable holds garbage memory; in JavaScript it safely holds undefined. Use === (strict equality) when checking for null to avoid JavaScript's loose comparison quirks.

let x;
console.log(x);          // undefined

let y = null;
console.log(y);          // null
console.log(y == null);  // true
console.log(y === null); // true (strict)
